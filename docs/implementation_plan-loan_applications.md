# Loan Applications Page — Implementation Plan

This plan outlines the steps to build the **Loan Applications** page (`/loan-applications`). As the core entity in the platform, this page will allow administrators and HR to view, manage, and process all loan applications.

## Proposed Changes

---

### Types & Data Layer

We will define the full entity schema and mock data for `LoanApplication` based on the architecture blueprint.

#### [NEW] [loan-application.ts](file:///Users/lnaguit/Desktop/code/loan-management-platform/app/types/loan-application.ts)
- Create types: `LoanApplication`, `LoanStatus`.
- Update `app/types/index.ts` to export this module.

#### [NEW] [loan-applications.json](file:///Users/lnaguit/Desktop/code/loan-management-platform/server/data/loan-applications.json)
- Create a mock dataset representing applications in various statuses (`Pending`, `Approved`, `Disbursed`, `Rejected`).
- Ensure the mock data maps correctly to existing Employee IDs (from `users.json`) and Loan Type IDs.

#### [NEW] [loan-applications.get.ts](file:///Users/lnaguit/Desktop/code/loan-management-platform/server/api/loan-applications.get.ts)
- Nitro endpoint to serve the initial mock data for the application seeder.

---

### State Management

#### [NEW] [useLoanApplications.ts](file:///Users/lnaguit/Desktop/code/loan-management-platform/app/composables/useLoanApplications.ts)
- Implement `useState` and `localStorage` persistence, following the established `useLoanTypes` pattern.
- Include methods: `load()`, `addApplication()`, `updateApplication()`, `deleteApplication()`, and `setApplications()`.
- Incorporate `useLogger()` for audit trailing (e.g., logging state changes like "Approved").

#### [MODIFY] [useDemoSeeder.ts](file:///Users/lnaguit/Desktop/code/loan-management-platform/app/composables/useDemoSeeder.ts)
- Update `seedAll()` to fetch `/api/loan-applications` and populate the local store.
- Update `resetAll()` to clear the loan applications.

---

### User Interface

We will build an interactive data table with status badges and an admin modal for creating or updating applications.

#### [NEW] [LoanApplicationModal.vue](file:///Users/lnaguit/Desktop/code/loan-management-platform/app/components/LoanApplicationModal.vue)
- Create a `UModal` (per your preference) featuring a `UForm` with Zod validation.
- Form fields:
  - **Employee**: Dropdown (populated via `useUsers()`)
  - **Loan Type**: Dropdown (populated via `useLoanTypes()`)
  - **Requested Amount**: Number input
  - **Term (Months)**: Number input
  - **Status**: Dropdown mapping to `LoanStatus`
- The modal will denormalize `employeeName` and `loanType` names upon submission.

#### [NEW] [loan-applications.vue](file:///Users/lnaguit/Desktop/code/loan-management-platform/app/pages/loan-applications.vue)
- Render a `UTable` listing all loan applications.
- **Columns**: Reference (`loanRef`), Employee, Type, Amount, Term, Date, Status (Color-coded badges).
- **Actions**: Edit, Delete, and quick-action status changes (e.g., "Mark as Approved").

#### [MODIFY] [default.vue](file:///Users/lnaguit/Desktop/code/loan-management-platform/app/layouts/default.vue)
- Add "Loan Applications" to the "Loan Operations" sidebar group, below "Loan Types".
